Advanced Writes: Writing with Dynamically Loaded Filters

This tutorial presumes that you have completed the Advanced Read: Installing Dynamically Loaded Filters tutorial and have set up your environment to use your dynamically loaded filters.
DataPipe objects can be used to write using Dynamically loaded filters. This tutorial will be using the Zstd dynamic filter as an example.
data = rand(1, 10000);
 
config = types.untyped.datapipe.Configuration(length(data));
blueprint = types.untyped.datapipe.BlueprintPipe(config);
blueprint.data = data;
The types.untyped.datapipe.properties.DynamicFilter property takes in an enumerated type ...datapipe.dynamic.Filter which is a hard-coded list of all listed registered filter plugins in HDF5.

Parameters

Some filter plugins allow for setting special configuration parameters to modify the filter's behavior. The DynamicFilter property type contains a modifiable parameters field which can be used to set your parameters. This is equivalent to setting the cd_values argument in HDF5. In the case of the Zstandard HDF5 plugin, the first (and only) array argument value indicates the compression level.
zstdProperty = types.untyped.datapipe.properties.DynamicFilter(types.untyped.datapipe.dynamic.Filter.ZStandard);
zstdProperty.parameters = 9; % compression level.

Writing

Writing is as normal with a BlueprintPipe. In a regular NWB file, you would simply replace the Pipe object with any dataset field and MatNWB will handle the low-level HDF5 files as normal.
blueprint.setPipeProperty(zstdProperty);
 
% Creates a H5 file of name `testzstd.h5` and overwrites. MatNWB handles
% this low-level call by default.
fid = H5F.create('testzstd.h5', 'H5F_ACC_TRUNC', 'H5P_DEFAULT', 'H5P_DEFAULT');
 
bound = blueprint.write(fid, '/data');
 
H5F.close(fid); % cleanup
The data is now compressed using Zstandard compression using a compression level of 9.
